Core Features

Everything You Need for Professional Surveillance

Built with a modern, scalable architecture for real-world CCTV deployments

Smart Auto-Discovery

Automatically scans your local subnet for devices with open RTSP port 554, then probes ONVIF capabilities on common HTTP ports — zero manual configuration required.

ONVIF RTSP Auto-Scan

Ultra-Low Latency Streaming

FFmpeg transcodes RTSP H.264 streams to MPEG-1 in real-time, delivering binary data via WebSockets to a canvas element rendered by JSMpeg — achieving latency under 0.5 seconds.

FFmpeg WebSocket JSMpeg

5 Premium Grid Layouts

Switch between 5 mathematically perfect 16:9 grid layouts — from compact 2x2 dual view to a full 4x4 matrix with 16 simultaneous feeds. All grids maintain exact widescreen proportions.

2x2 3x3 4x4 Spotlight

ONVIF PTZ Control

Control Pan, Tilt and Zoom via an interactive D-Pad with three speed levels (Slow, Medium, Fast) and automatic failsafe stop after 500ms. Full ONVIF SOAP protocol compliance.

Pan/Tilt Zoom 3 Speeds

Live Drag and Drop Reorder

Rearrange camera positions in real-time by dragging cards within the grid. New order is instantly applied and persisted to the local database — no page reload required.

HTML5 DnD Persistent

Instant Snapshots

Capture pixel-perfect screenshots from any live stream at any moment — no black frames. Download camera snapshots directly from the browser with a single click.

Canvas API PNG Export

Resource Saver Mode

Activate Eco Mode to pause all active video streams and reduce CPU/bandwidth usage drastically — perfect for long-duration monitoring sessions.

Eco Mode CPU Saver

i18next EN PT-BR

Quick Start

Up and Running in Minutes

Choose between the automated Windows installer or manual setup via CLI

Windows Quick Start

Automated .bat scripts — just double-click

  1. 1
    Clone the repository git clone https://github.com/BrunoKappi/vms-monitor.git
  2. 2
    Run scripts/setup-vms.bat

    Double-click scripts/setup-vms.bat — installs Node.js dependencies, checks FFmpeg, and configures the .env file automatically.

  3. 3
    Launch with scripts/start-vms.bat

    Double-click scripts/start-vms.bat — starts both servers and opens the dashboard in your browser automatically at http://localhost:42100.

  4. 4
    Stop with scripts/stop-vms.bat

    Double-click scripts/stop-vms.bat to gracefully shut down all servers and free ports 42100, 42200, and 42300.

Manual CLI Setup

Node.js v18+ and FFmpeg required

Prerequisites 
# Install Node.js v18+ from https://nodejs.org/
# Install FFmpeg from https://ffmpeg.org/download.html
# Add ffmpeg/bin to your system PATH
Installation 
git clone https://github.com/BrunoKappi/vms-monitor.git
cd vms-monitor
npm install
npm run install-all
Run 
npm run dev
# Open: http://localhost:42100
